    __setRegionsList(self) - формирует список вторых частей url для переходов для парсинга по регионам. Включена в состав конструктора
    searchingValues(self) - осуществляет парсинг по каждому региону посредством создания экземпляра класса regionParser для каждого региона и дальнейшего получения данных 
        по каждому региону внутри этого класса

    class regionParser - собирает url, делает запрос и получает необходимые данные о стоимостях бензина. Внутри него:
        __soupSetting(self) - определяет материал для парсинга страницы. Так как данные на этой странице подгружаются динамически, выбран способ загрузки страницы через Selenium,
            далее при помощи self.__driver.page_source исходный код страницы передается в парсер bs. В метод добавлены неявные ожидания загрузки body и DOM. 
        __priceParsing - собирает информацию о ценах и заносит цены в список self.__petrol. 
        Оба методы включены в конструктор и вызываются автоматически при создании объекта

    class privatePetrol() - планируется собирать все данные в список __petrolPrice. Для этого посредством __parsing(self), запускаемого изконструктора, переходит по прямой ссылке 
        на страницу региона. Данные для формирования ссылки берет из regions.json. Метод __parsing(self) сначала пытается забрать исходный код страницы при помощи requests, 
        в случае неудачи пробует сделать то же самое посредством Selenium. В код добавлены задержки для моделирования человеческого поведения, иначе сайт блокирует IP на сутки.
        Вложенный класс toJson() забирает данные - список списков __petrolPrice родительского класса, создает файл json с текущей датой и помещает данные в него.
